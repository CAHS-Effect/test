How it works
All articles, writers, and issues are stored in a data/db.json file in your GitHub repository. Public pages load data from GitHub's CDN — no server needed. The CMS writes back via the GitHub API using a Personal Access Token (PAT) you provide below.

Setup steps:
1. Create a GitHub repo (public recommended for free GitHub Pages hosting).
2. Go to github.com → Settings → Developer settings → Personal access tokens → Fine-grained tokens.
3. Create a token with Contents: Read and Write permission for your repo.
4. Enter your repo details and token below, then click Connect & Initialise.
5. Deploy your site files to the same repo and enable GitHub Pages.

Note: Your PAT is stored only in this browser's localStorage — it is never committed to the repo.
